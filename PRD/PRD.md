# 인건비 계산기 — 제품 요구사항 정의서 (PRD)

**문서 버전:** 1.0  
**최종 반영일:** 2026-04-28  
**대상 코드:** `index.html` (단일 페이지)

---

## 1. 제품 개요

### 1.1 목적
정부지원금(천 원 단위)과 기업 유형을 입력하고, 기존·신규 참여 인력 정보를 바탕으로 **인건비 총괄(표1)** 및 **인건비 세부(표2)**를 산출하는 웹 기반 참고용 계산기다. 과제별 공고·작성지침과 다를 수 있음을 사용자에게 명시한다.

### 1.2 주요 사용자
사업비·인건비 편성을 검토하는 담당자(기업/기관).

### 1.3 범위
- 브라우저에서 동작하는 정적 HTML + 인라인 스크립트.
- 별도 서버·DB 없음, 입력 데이터는 세션 메모리에만 존재.

---

## 2. 핵심 비즈니스 규칙(요약)

| 항목 | 내용 |
|------|------|
| 정부지원금 | 기존 인력 **현금**에만 배치 |
| 기업부담금(현금) | 신규 인력 **현금**에만 배치 |
| 기업부담금(현물) | 기존 인력 **현물**에만 배치 |
| 기존 인력 | 1인당 **현금 또는 현물 중 하나**만 (동시 배치 불가) |
| 정책 | 기업 유형별 `POLICY`: 정부지원 비율, 기업부담 내 현금 비율 |

표1(인건비 총괄)은 `buildTable1(govFund, policy)`로 **총사업비 역산 → 기업부담 총액 → 현금/현물 분해**를 수행한다.

## 3. 화면 구성

### 3.1 입력 영역
- 정부지원금(천 원), 기업 유형(대기업/중견/중소/공기업).
- 기존 인력 추가 / 신규 인력 추가, **출력** 버튼.
- 인력 테이블: 성명, 직위, 구분(정규직·계약직), 총 연봉(천 원), 참여 개월 수, 복사·삭제, 드래그 순서 변경.

### 3.2 결과 영역
- **좌측:** 결과 본문(플레이스홀더 / 표1 / 표1+표2).
- **우측:** 오류·경고·정상 메시지 스택.

### 3.3 상단 안내
- 제목 하단 가이드 문구: 공고문·작성지침 확인 권고.

## 4. 기능 요구사항 (현재 구현 기준)

### 4.1 입력 검증 (`validateInputs`)
- 정부지원금: 유한수이고 0보다 커야 함.
- 기업 유형: `POLICY`에 정의된 값이어야 함.
- 참여인력: **최소 1명 이상**.
- 인력별: 성명·직위·구분·연봉(>0)·참여개월(1~12) 필수·유효.

검증 실패 시: 우측에 오류 메시지, 결과 영역은 `"입력값 오류를 먼저 수정해 주세요."` 플레이스홀더.

### 4.2 계산 실행 (`buildCalculation`)
- 입력이 통과하면 전체 배분 로직 수행.
- **조건:** `표1의 정부지원금 > 기존 인력 참여개월 반영 연봉 합계`  
  - 오류 메시지: `정부지원금 총액이 기존 인력 총계보다 큰 경우 계산할 수 없습니다.`
  - **표2(세부)는 생성하지 않음.** 다만 **표1만 결과 영역에 출력**해야 함 (아래 4.4).
- 동일 조건일 때 표2 정합성 검사·0원 배분·참여율 관련 **추가 오류·경고는 표1 전용 모드에서 생략**하여, 사용자에게 불필요한 연쇄 오류가 쌓이지 않도록 한다.

### 4.3 정상 완료 시
- 표1 + 표2 + 하단 **안내** 블록(`footer-note`) 출력.
- 우측 메시지에 성공/경고 등 표시.

### 4.4 표1 전용 출력 (오류가 있어도 표1 표시)
**트리거:** 위 4.2의 “정부지원금 > 기존 인력 총계” 오류 **단독으로** `table1Only === true`인 경우.

**동작:**
- 우측: 해당 오류 메시지 유지.
- 결과 영역: **인건비 총괄(표1)** + 하단 안내. 표2 없음.
- 구현: `renderTable1Only(result)`, HTML 생성은 `getTable1Html(table1)` 공통 사용.

### 4.5 기타 계산 오류
`table1Only`가 아닌 오류가 있으면 결과 영역은  
`계산 조건을 충족하지 않아 결과 표를 생성할 수 없습니다.` 플레이스홀더 (기존 동작).

## 5. UI 요구사항 (본 차수 반영)

### 5.1 플레이스홀더와 안내 박스 간격
- 결과 영역에서 `renderResultPlaceholder`로 그려지는 구조:  
  상단 `.empty`(점선 박스 메시지) + 하단 `.footer-note`(안내).
- **요구:** 두 박스 사이 세로 간격 **20px**.  
- 구현: `.result-main .empty + .footer-note { margin-top: 20px; }`
- 표 아래 안내와의 간격은 별도 규칙(인접 형제 선택자로 플레이스홀더에만 적용).

## 6. 명시적으로 포함하지 않는 사항 (히스토리)

다음은 대화 과정에서 **요청되었다가 사용자 요청으로 원복**된 동작이며, **현재 코드에는 없음**.

- **정부지원금만 입력**한 상태(인력 미입력 또는 미완성)에서도 검증을 우회하고 표1만 항상 출력하는 흐름.

현재는 **참여인력 검증을 통과해야** 본 계산·표1 전용 분기까지 진행된다.

## 7. 비기능 요구사항

- **단위:** 금액은 천 원, 내림(`floorKrwToThousand`) 적용 구간 존재.
- **반응형:** 좁은 화면에서 결과 레이아웃 1열 전환.
- **접근성:** 기업 유형 도움말 아이콘 등 기본 상호작용 유지.
